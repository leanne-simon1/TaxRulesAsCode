Rules explorer

Everything on this page is read live from the rules engine's metadata API — labels, plain language documentation, legislative references, formula code and dated policy parameters. Nothing is hard-coded here, so this view can never drift from the rules that actually run. This is the view a tax technical specialist can check without reading Python.

Policy parameters dated values — editable without code

Values that change over time (rates, thresholds and, later, objection time limits) live in YAML parameter files with a start date per value. Updating policy means adding a dated row — no formula changes, and CI tests still gate the change.
